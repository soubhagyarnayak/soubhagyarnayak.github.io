Moving Blog to Staticsite

2021-07-18
3 minute read

Two years back, I have moved my blog from wordpress.com to hosted wordpress on Godaddy. The subscription for 2 years is going to end soon. So I was in a dilemma whether to renew my subscription or try some alternative. After much thought, I have decided to move my blog to a static website hosted on Netlify.

Why to use static website for a blog?

Wordpress is the most popular tool for hosting web pages and blogs. It is very powerful. It has a rich set of plug-ins and themes. I have been using Wordpress for a long time. So initially I had thought to move the blog to my droplet on DigitalOcean. I use the droplet to host my web services and web sites. But setting up and maintaining wordpress is a lot of work. Also I am writing the blog mostly for myself. I use it to reflect on ideas, document things for later use and maintain a weblog of my work. So the rich support provided by wordpress for SEO and comments etc are not needed for me. At the same time, I need to maintain and track my changes via git. Static websites generated from markdown files are fitting to the two requirements quite nicely. There are other benefits as well like low maintenance, security and low hosting cost for static websites.

How to generate the static website?

So I am going to write my blog posts in markdown. I will check-in those files to my github repo. There a number of tools available to generate static web pages from markdown files. I have decided to go with pelican. Most of the tools have similar capabilities. Pelican is written in python and is quite robust and mature.

Setting up Pelican

  1. First we will need to install pelican with markdown support. bash python -m pip install "pelican[markdown]"

  2. Next we need to create a directory for our blog and create a pelican based website.
    bash mkdir myblog && cd myblog && pelican-quickstart
    pelican-quickstart is an interactive command which will ask for user inputs for customization.

  3. Once the pelican setup is done, we can go to the content directory and add markdown files. Each markdown file will corresponds to a post. Inside the markdown file, we will start with a set of attributes to specify the metadata. Then we will specify our content in the markdown.

  4. The command pelican content generates the static web pages from markdown files and puts them into the output directory.
    bash pelican content

  5. Then I publish the directory to Netlify and have my blog up. To view the generated site on local machine, one can run pelican –listen command. This will start serving the static pages locally on port 8000. So one can visit https://localhost.com:8000 to view the website.

Pelican Themes

Themes help in customizing the UI. Pelican has rich support for custom themes. One can visit the pelican themes page to view the available options. One can also create his own theme. To install a new theme, one has to pull the theme from its git repo and install it via pelican-themes command. Please refer to the documentation for all using pelican-themes command.

1pelican-themes -l # list installed themese
2pelican-themes -i ./themes/brutalist # install a theme from a directory
3pelican-themes -r brutalist # remove an installed theme

Every theme has some custom settings which are specified in the README file of the theme. Generally one has to update pelicanconf.py file to update the settings. To apply a theme to your site, you will have to add/edit the THEME value.

1THEME = 'themes/brutalist'
