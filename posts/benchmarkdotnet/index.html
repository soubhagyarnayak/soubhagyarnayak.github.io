BenchmarkDotNet

2020-11-01
3 minute read

While developing software, we often have the need to benchmark some implementations and compare the performance. Every benchmarking task has common boiler plate code which is tedious and repetitive. BenchmarkDotNet helps in avoiding this boiler plate code and focus on the code which needs to be benchmarked.

How to do benchmarking using BenchmarkDotNet

I generally make a standalone app to run the benchmarks. The standalone app needs to take a dependency on BenchmarkDotNet nuget package. The code shown below demonstrates how to benchmark StringBuilder vs string concatenation performance. This code sample can be used as a template for any types of benchmarking.

Then we need to build code. While building, please make sure to use Release option and not Debug. Once the exe is built, we can go to the bin\Release directory and run the exe from command prompt. It will take some time to run the code and derive statistically significant data points. The best part is that one need not write any boiler plate code or understand deep statistics concept to benchmark code. BenchmarkDotNet takes care of this.

The following image shows the data shown in command prompt. !BenchmarkDotNet output on command line

BenchmarkDotNet also stores the result in the bin\Release\BenchmarkDotNet.Artifacts directory. Depending upon the configuration, BenchmarkDotNet will generate CSV, MarkDown, HTML and even R plots in the artifacts directory.

Advanced Concepts

Bringing variations in input data

If we see the above result, then we will observe that the performance of string concat comparable to using StringBuilder. To get more insights we can bring more variations in the input. In the following altered code, we are changing the number of strings to concat. BenchaMarkDotNet helps us in doing this pretty easily.

 1//The following attribute specified parameters for benchmark runs
 2[Params(3, 7, 21, 50, 100)]
 3public int NumberOfSubParts;
 4//GetRandomStrings has been altered to use the parameter value
 5private string[] GetRandomStrings()
 6{
 7    string[] result = new string[NumberOfSubParts];
 8    for(int i=0; i<NumberOfSubParts; i++)
 9    {
10        result[i] = RandomString(random.Next(7));
11    }
12    return result;
13}
