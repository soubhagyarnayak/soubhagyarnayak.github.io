Dropwizard

2021-01-23
2 minute read

In Java world, the developers have lots of framework to choose to develop web services and web sites. But most of the frameworks are bulky as they try to support a lot of features. Dropwizard is a lightweight opinionated framework for developing RESTful web services in Java.

Setting up

Dropwizard is easy to setup using Maven archetype. One can create a Dropwizard project by the following command

1mvn archetype:generate 
2    -DarchetypeGroupId=io.dropwizard.archetypes 
3    -DarchetypeArtifactId=java-simple 
4    -DarchetypeVersion=[Version of DropWizard]

Build and Deploy

Generally Dropwizard project creates a fat JAR file during build. The single JAR file has all the required class files. The following command can be used to build the JAR file

1mvn clean package

The JAR file can be run easily by the following command.

1java -jar target/<output>.jar server [customTemplateFile].yml

Basic Concepts

Configuration class

A project can have a class extending io.dropwizard.Configuration. The configuration class specifies the environment specific parameters. The parameters are specified in a YAML configuration file to deserialized to the configuration class at runtime. Dropwizard uses Jackson to deserialize the configuration file.

Application class

A project needs to have an application class extending io.dropwizard.Application. This application class contains the Main function which is the entry point of the application. There are 2 other important methods

  1. initialize: to configure application before run.
  2. run: to run the application.

Resource class

Dropwizard follows RESTful model. So each resource class specifies a resource and is associated with a URI template with @Path annotation. The resource class uses JAX-RS annotations to specify methods (GET/POST), input and output.

Representation class

Representation class is a simple POJO used to specify the data model for API calls. Jackson annotations are used to aid in serialization.

Registering resources

Resources are registered in the run method of application class. Registering involves creating an instance of a Resource class and registering it to the environment by the following API call.

1environment.jersey().register(customResourceInstance);

Health check

Dropwizard has good support for creating health check APIs. First we need to create a class extending com.codahale.metrics.health.HealthCheck. The health check class needs to implement the check method to specify the health check logic. Then in the run method of application class, the health check class needs to be instantiated and registered via the following API call.

1environment.healthChecks().register("test", customHealthCheckInstance);

Out of box resources

Resource Name Default URI
Admin Page /
Metrics /metrics
Thread dump /threads
Health check /healthcheck

Included Libraries

The framework is opinionated as it chooses a small set of popular packages to have the desired features. The following packages are used

Package Feature
Jetty Http Server
Jersey REST/JAX-RS
Jackson JSON parsing
Dropwizard Metrics Metrics
LogBack and slg4j Logging
Hibernate Validator Input validation
Apache HTTP Client and Jersey Client HTTP client
JDBI RDBMS interaction
Liquibase Database schema handling
Freemarker and Mustache templating
Joda Time Date and time management
